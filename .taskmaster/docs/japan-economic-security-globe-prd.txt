# PRD: Japan Economic Security Globe Phase 0 MVP

## Summary

Build the Phase 0 public MVP for `japan-economic-security-globe`, a public-interest semantic intelligence service for people in Japan. The product explains what Japan depends on globally and how those dependencies flow back into rice, water, electricity, daily life, defense, and semiconductors.

The app is a Japan-first operating picture. The first screen is not a globe demo and not a generic dashboard. It is a full-screen Japan map canvas with supporting navigation, comparison, and evidence surfaces. Global routes and foreign countries appear only as explanatory context for Japan. Phase 0 must be free, public, source-linked, and built around a clean semantic data model that can later migrate to OWL/RDF/SPARQL infrastructure.

## Goals

1. Create a working local Next.js App Router application.
2. Build a visually strong public homepage with a full-screen Japan operations map, supporting global context, and an evidence graph.
3. Use Energy as the default story lens and include complete thin slices for Rice, Water, Defense, and Semiconductors.
4. Model all MVP content as a canonical semantic graph rather than per-component hard-coded data.
5. Add local seeded JSON data with provenance fields aligned to future `prov:wasDerivedFrom`.
6. Add initial Turtle ontology stubs and SPARQL query examples.
7. Add README explaining the project concept, semantic-web value, MVP scope, and future OWL/RDF/SPARQL plan.

## Non-Goals

1. Do not implement authentication.
2. Do not implement billing or paid features.
3. Do not implement a real SPARQL endpoint.
4. Do not implement live data ingestion.
5. Do not add RDS or external database dependencies.
6. Do not fully model domestic trucking, warehouse, or distribution logistics.
7. Do not create five disconnected mini apps.

## Tech Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- MapLibre GL JS for the main Japan map canvas
- Lightweight SVG or GeoJSON-based overlays for semantic regions and supporting mini-views
- react-force-graph or a similar graph component for evidence graph
- Vitest for semantic selectors and SPARQL preview tests
- Local JSON/TTL seed files
- Cloudflare-only compatible architecture for future deployment

## Product Framing

The service is explicitly for people in Japan. Foreign countries and routes are explanatory variables, not primary service regions. Every selected country, route, port, facility, policy, budget, or source should ultimately answer why it matters to people in Japan.

Phase 0 is `Japan civic dependency service`.

Phase 1 later adds `Japan strategic neighborhood layer` while keeping the audience unchanged.

Phase 2 later adds institutional paid features such as private workspaces, alerts, API/data access, internal source integration, scenario analysis, and custom ontology build services.

## Required Phase 0 UX

1. A top action bar shows the product name, active story lens, map mode, shareable state, and high-level actions.
2. A left navigation rail contains story lenses, semantic presets, and operational filters rather than only visual theme buttons.
3. Energy is the default active story lens, but the navigation model must work for all themes without redesign.
4. The main hero area is a full-screen Japan operations map canvas.
5. The initial viewport is fixed on Japan and must not auto-fit to a selected route on first load.
6. The initial map mode is `point` or `cluster`; route mode is a second-step analysis view, not the default first impression.
7. Zooming out or switching mode reveals global supplier countries, chokepoints, and sea-lane routes as supporting context.
8. The bottom comparison grid lists routes, observations, facilities, and policy signals in a comparable operational table.
9. The right evidence drawer shows summary, why it matters for Japan, linked flows, source documents, related entities, and SPARQL preview.
10. Evidence graph connects source documents, policy documents, budget lines, laws, organizations, facilities, products, observations, and flows.
11. Clicking a country, route, chokepoint, port, terminal, refinery, reservoir, prefecture, observation, policy document, budget line, or source document updates the detail view.
12. Foreign country detail windows show flag, small map/geographic cue, relation to Japan, flows to Japan, key sources, and related themes. They must not become generic country profiles.

## Navigation and Selection Architecture

The navigation model should follow an operating-picture pattern rather than a dashboard-card pattern.

Primary areas:

1. Action bar
2. Navigation rail
3. Map canvas
4. Context drawer
5. Comparison grid

Action bar responsibilities:

- show current lens and map mode
- expose global search or command entry
- expose share/copy-link behavior
- keep the current state legible without opening side panels

Navigation rail responsibilities:

- choose story lens by public question, not by raw data type
- expose semantic presets such as `中東輸送路`, `コメ価格`, `水ストレス`, `FY2026防衛予算`, `半導体供給`
- expose filters and counts
- remain compact enough that the map canvas stays primary

Map canvas responsibilities:

- remain the main operational surface
- default to a stable Japan viewport
- support point, cluster, choropleth, route, and static analysis modes
- reveal global context only when zoom or mode indicates the user asked for it

Context drawer responsibilities:

- explain the selected object
- connect to evidence, source documents, and semantic relations
- avoid dominating the initial screen

Comparison grid responsibilities:

- support triage, ranking, and comparison
- synchronize with map selection and URL state
- feel like an operational list rather than a generic data table

The selection model should follow a scenario-select pattern:

- users choose a public question or story lens
- the app binds that lens to a semantic preset
- the preset controls which entities, flows, observations, and sources become primary
- future data refreshes should update the preset automatically without changing the navigation structure

## Seed Data Requirements

Create local seed data under `data/seed/`:

- entities.json
- flows.json
- observations.json
- sources.json

Every item must include source labels and source URLs. All facts must include provenance fields that can later map to `prov:wasDerivedFrom`.

Theme slices:

1. Energy: Japan linked to oil, LNG, and coal supplier countries; Hormuz and Malacca; ports/terminals/refinery examples; electricity/fertilizer impact.
2. Rice: rice price, stockpile, import, and policy signal observations connected to energy/fertilizer dependency.
3. Water: selected prefectures or reservoir systems with water stress/drought examples.
4. Defense: sample FY2026 budget categories linked to capability areas and source documents.
5. Semiconductors: Japan linked to Taiwan, South Korea, Netherlands, United States, and China.
